Package: org.openstreetmap.josm.gradle.plugin.config

[Groovy] Class JosmManifest

    • org.openstreetmap.josm.gradle.plugin.config.JosmManifest

    • The info that will be written into the manifest file of the plugin *.jar

      Most of the attributes are corresponding to the ones listed in the documentation for developing JOSM plugins. But note that some of them have been renamed a bit to better express what each attribute is good for.

      The properties used by the Ant build (e.g. plugin.author or plugin.description) are automatically picked up by both Ant and Gradle, if you define them in your {@gradle.properties} file and load them into your build.xml file using the following snippet: 

       <!-- edit the properties of this plugin in the file `gradle.properties` -->
 <property file="${basedir}/gradle.properties"/>

      The detailed documentation for each field below tells you, which one corresponds to which Ant/Gradle property and to which attribute in the MANIFEST.MF file.

      • Property Detail

        • String author

          The author of the plugin.

          Default value:
          the value of property plugin.author or null if that property is not set.
          Influenced MANIFEST.MF attribute:
          Author

        • boolean canLoadAtRuntime

          Determines if the plugin needs a restart after installation. true if no restart is required, false otherwise.

          Default value:
          the value of property plugin.canloadatruntime or false if that property is not set.
          Influenced MANIFEST.MF attribute:
          Plugin-Canloadatruntime

        • String description

          The description of what the plugin does.

          Default value:
          the value of property plugin.description or null if that property is not set.
          Influenced MANIFEST.MF attribute:
          Plugin-Description

        • String iconPath

          Path to the logo of the plugin. Relative to the root of the released jar-file, so that it can be loaded via getClass.getResource().

          Default value:
          the value of property plugin.icon or null if that property is not set.
          Influenced MANIFEST.MF attribute:
          Plugin-Icon

        • boolean loadEarly

          This can be set to true, when the plugin should load before the GUI classes of JOSM.

          Default:
          The value of property plugin.early or false if that property is not set.
          Influenced MANIFEST.MF attribute:
          Plugin-Early

        • Integer loadPriority

          A number indicating the order in which the plugins should be loaded. Lower numbers first, higher numbers later, then the plugins with this field set to null.

          Default:
          The integer value of property plugin.stage or null if that property is not set.
          Influenced MANIFEST.MF attribute:
          Plugin-Stage

        • String mainClass

          The full name of the main class of the plugin

          Default:
          the value of property plugin.class or null if that property is not set.
          Influenced MANIFEST.MF attribute:
          Plugin-Class

        • String minJosmVersion

          The minimum JOSM version with which the plugin is compatible.
          This field is required!

          Default:
          the value of property plugin.main.version or null if that property is not set.
          Influenced MANIFEST.MF attribute:
          Plugin-Mainversion

        • final Set<String> pluginDependencies

          A collection of the names of all JOSM plugins that must be installed for this JOSM plugin to work

          Default:
          the value of property plugin.requires split at every semicolon (do not rely on the order, as it is not necessarily maintained) or null if that property is not set.
          Influenced MANIFEST.MF attribute:
          Plugin-Requires

        • URL website

          A URL pointing to a web resource describing the plugin.

          Default:
          The value of property plugin.link as URL (might error out with a MalformedURLException on malformed URLs), or null if that property is not set.
          Influenced MANIFEST.MF attribute:
          Plugin-Link

      • Method Detail

        • Map<String, String> createJosmPluginJarManifest()

          Returns a map containing all manifest attributes, which are set. This map can then be fed into Manifest.attributes. That's already done automatically by the gradle-josm-plugin, so you normally don't need to call this yourself.

        • void oldVersionDownloadLink(int minJosmVersion, String pluginVersion, URL downloadURL)

          Add a link to an earlier release of the plugin, that is compatible with JOSM versions, with which the current version is no longer compatible.

          Influenced MANIFEST.MF attribute:
          ([1-9][0-9]*)_Plugin-Url (Any attribute that matches this RegEx. The number in the beginning is determined by the parameter minJosmVersion.)
          Parameters:
          minJosmVersion - the minimum JOSM version with which the linked plugin is compatible
          pluginVersion - the version number of the linked plugin
          downloadURL - the URL where the linked plugin can be downloaded from

        • void translatedDescription(String language, String translatedDescription)

          Adds a translation of the plugin description for a certain language.

          Parameters:
          language - the language abbreviation (e.g. de_AT or es)
          translatedDescription - the description in the language given by the language parameter
